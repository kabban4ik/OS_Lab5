# CONTRIBUTING

Дякую, що хочете зробити внесок у цей проєкт!

## Як почати

1. Зробіть *fork* репозиторію.
2. Створіть нову гілку для своєї зміни:
   ```
   git checkout -b feature-name
   ```
3. Внесіть зміни та зробіть атомарні коміти.

## Формат комітів

- Пишіть короткий та зрозумілий commit message.
- Використовуйте формат:
  - feat: для нових можливостей
  - fix: для виправлення помилок
  - docs: для документації
  - style: зміни в коді без логіки
  - refactor: покращення коду
  - test: додавання тестів

## Pull Requests

1. Переконайтесь, що ваша гілка оновлена відносно `main`.
2. Відкрийте *Pull Request* з чітким описом:
   - Що було змінено?
   - Навіщо зміна?
3. Чекайте перегляду (review).

## Стандарти коду (якщо це програмний проєкт)

- Пишіть чистий та зрозумілий код.
- Уникайте дублювань.
- Використовуйте зрозумілі назви змінних.

## Питання

Якщо у вас є питання — створіть `Issue` у GitHub.

Успіхів!
